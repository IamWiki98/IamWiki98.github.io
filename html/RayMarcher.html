Ray-Marcher

During Year 2 at Breda University of Applied Sciences, I decided to learn the ray-marching algorithm and I implemented shaders using ShaderToy. The algorithm seems to work as ray-tracing does but is actually completely different. Every shape or visual effect is done entirely by mathematical algorithms. This page mostly represents my mathematical skills.


Fluid Tank

This shader is my first experience with Ray-Marching. I was mostly copying code and analyzing it but I am still proud of the result. There are two sections implemented with different algorithms in an infinite cylinder and the camera is following the curvature of the cylinder.

My first try with Ray-Marching

I experimented with the techniques I learned from the shaders I analyzed. I created 10+ shaders using different methods I found in ShaderToy. For the examples above: I create a variety of shapes and rotate them by combining the result in different ways. Ray-Marching lets you combine shapes to create fluid-like simulations.

Fractions

I found how to use fractions in ray-marching and with them, I created infinite 'particles'. At this point, I felt confident enough to create something unusual by spending more time on a single shader. All the shaders above were implemented in a couple of hours.

My best shader (Warning: This is GPU performance heavy)

This shader was implemented by combining many different shaders into one. It wasn't as simple as copy/paste. I needed to figure out how they work in order to implement the same tweaked code in my shader. Finally, the result I achieved looked like a movie scene.
